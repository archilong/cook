# 菜谱列表缩略图展示设计

日期：2026-07-17

## 1. 背景

移动端个人菜谱列表和家庭菜谱列表原本在列表项顶部展示菜谱主图，图片占据整行宽度。用户希望列表页只显示缩略图，把图片放在列表内容右侧，点击进入菜谱详情后再显示大图。

这次调整聚焦菜谱列表的信息密度和浏览体验，不改变菜谱详情页、图片上传流程或后端图片数据结构。

## 2. 目标

1. 个人菜谱 Tab 列表中，菜谱图片不再作为顶部大图展示。
2. 家庭 Tab 首页中，家庭共享菜谱图片不再作为顶部大图展示。
3. 家庭菜谱二级列表中，菜谱图片不再作为顶部大图展示。
4. 列表项采用“文字内容在左，图片缩略图在右”的布局。
5. 菜谱详情页和家庭菜谱详情页继续显示大图。
6. 没有主图的菜谱继续只展示文字信息，不额外显示空白占位图。

## 3. 非目标

- 不改变菜谱详情页的大图展示。
- 不修改图片上传、图片裁剪或图片存储逻辑。
- 不改变菜谱列表的数据来源或接口。
- 不新增图片懒加载、缓存策略或骨架屏。
- 不调整清单/任务页中的订单图片展示。

## 4. 页面范围

需要调整的列表页：

- `mobile/app/(tabs)/recipes.tsx`：个人菜谱 Tab 首页。
- `mobile/app/(tabs)/family.tsx`：家庭 Tab 首页中的当前家庭共享菜谱列表。
- `mobile/app/families/[id]/recipes.tsx`：家庭菜谱二级列表。

保持不变的详情页：

- `mobile/app/recipes/[id].tsx`：个人菜谱详情页，继续显示大图。
- `mobile/app/families/[id]/recipes/[recipeId].tsx`：家庭菜谱详情页，继续显示大图。

## 5. 列表项布局

每个有图片的菜谱列表项使用横向布局：

- 外层仍是当前列表项卡片或可点击区域。
- 内层使用横向容器。
- 左侧文字区域 `flex: 1`，展示标题、作者、共享人、步骤数、标签等当前已有信息。
- 右侧图片区域为固定尺寸缩略图。
- 缩略图使用圆角和浅色底色，保持与当前菜谱图片视觉风格一致。
- 缩略图使用 `resizeMode="cover"`，避免长图或宽图在小区域里出现过多留白。

推荐缩略图尺寸：

- 宽度：`96`
- 高度：`84`
- 圆角：`10`

没有图片时，不显示缩略图，文字区域自然占满可用宽度。

## 6. 交互行为

列表项点击行为保持不变：

- 个人菜谱列表项点击进入 `/recipes/[id]`。
- 家庭 Tab 首页菜谱列表项点击进入 `/families/[id]/recipes/[recipeId]`。
- 家庭菜谱二级列表项点击进入 `/families/[id]/recipes/[recipeId]`。

图片本身不新增独立点击行为，整个列表项仍然是进入详情页的点击目标。

## 7. 验证标准

运行时验证应覆盖：

1. 打开个人菜谱 Tab，带主图的菜谱在列表右侧显示缩略图，不再显示整行大图。
2. 打开家庭 Tab 首页，当前家庭共享菜谱在列表右侧显示缩略图，不再显示整行大图。
3. 打开家庭菜谱二级列表，带主图的菜谱在列表右侧显示缩略图，不再显示整行大图。
4. 点击个人菜谱进入详情页后，详情页仍显示大图。
5. 点击家庭菜谱进入详情页后，详情页仍显示大图。
6. `npm run typecheck` 通过。

## 8. 实施记录

本设计已按用户反馈完成实现。根因是家庭 Tab 首页 `mobile/app/(tabs)/family.tsx` 也有一处独立的家庭共享菜谱列表，最初只修改了个人菜谱列表和家庭菜谱二级列表，遗漏了该入口。最终三处列表入口均已改为右侧缩略图，详情页大图保持不变。
